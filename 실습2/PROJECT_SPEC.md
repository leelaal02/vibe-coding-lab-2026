# 날씨 조회 웹앱 프로젝트 명세서

## 1. 프로젝트 개요
대한민국 기상청(KMA) API를 활용하여 전국 각 지역의 날씨 예보 정보를 조회하고 표시하는 웹 애플리케이션입니다.

## 2. 기술 스택
- **프레임워크**: Next.js (버전 16 이상)
- **라이브러리**: React, TypeScript
- **스타일링**: Tailwind CSS
- **HTTP 클라이언트**: Axios

## 3. 외부 API 연동

### 3.1 날씨 예보 API
- **엔드포인트**: `https://apihub.kma.go.kr/api/typ01/url/fct_afs_dl.php`
- **매개변수**:
  - `reg`: 예보구역코드 (REG_ID)
  - `tmfc`: 발표시각 (선택 사항)
  - `disp`: 출력 모드 (1)
  - `help`: 도움말 플래그 (1)
  - `authKey`: 발급받은 인증번호
- **주요 데이터 필드**:
  - `REG_ID`: 예보구역코드
  - `TM_EF`: 발효시각 (예보 시간)
  - `TA`: 기온 (°C)
  - `ST`: 강수확률 (%)
  - `SKY`: 하늘상태코드 (DB01: 맑음, DB02: 구름조금, DB03: 구름많음, DB04: 흐림)
  - `PREP`: 강수유무코드 (0: 없음, 1: 비, 2: 비/눈, 3: 눈, 4: 소나기)
  - `TM_FC`: 발표시각

### 3.2 예보구역코드 조회 API
- **엔드포인트**: `https://apihub.kma.go.kr/api/typ01/url/fct_shrt_reg.php`
- **매개변수**:
  - `tmfc`: 0
  - `authKey`: 발급받은 인증번호
- **목적**: `REG_ID`를 사용자 친화적인 `REG_NAME`으로 변환하여 표시하기 위함.

## 4. 주요 기능
- **지역 날씨 검색**: 지역명으로 날씨를 검색 (내부적으로 `REG_ID` 매핑).
- **예보 표시**: 기온, 강수확률, 하늘상태를 포함한 현재 및 향후 예보 정보 제공.
- **사용자 친화적 명칭**: UI에서 코드값(`REG_ID`) 대신 지역명(`REG_NAME`) 출력.

## 5. UI/UX 디자인 목표
- **프리미엄 디자인**: 풍부한 색상 팔레트와 현대적인 타이포그래피를 사용한 고품질 비주얼.
- **반응형 레이아웃**: 모바일과 데스크톱 모두에서 최적화된 경험 제공.
- **상호작용 요소**: 날씨 아이콘 및 화면 전환 시 부드러운 애니메이션 적용.

## 6. 구현 참고 사항
- API 인증 및 베이스 URL 관리를 위한 Axios 인터셉터 설정.
- API 응답 파싱을 위한 TypeScript 인터페이스 정의.
- 기상청의 쉼표 구분/텍스트 기반 응답 형식을 처리하기 위한 유틸리티 함수 구현.
